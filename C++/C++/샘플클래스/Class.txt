1.객체지향 프로그래밍 (OOP, Object - Oriented Programming)
- 객체지향 프로그래밍에서는 객체가 중심
- 객체의 상태(State) , 행동 (Behavior)를 구체화시키는 작업
- 클래스는 객체를 만들어 내는 설계도 

2. 클래스 (Class)
- 객체를 정의하는 틀, 설계도 
- 개발자는 클래스를 이용해서 여러 객체를 만들어서 사용 (인스턴스 )
- 클래스는 객체의 상태를 나타내는 필드 (변수 ) 와 객체의 행동을 나타내는 메소드(함수) 로 구성됨

- 필드 : 클래스에 포함된 변수
- 메소드 : 클래스에 포함된 함수

3. 인스턴스 (Instance)
- 설계도 (클래스 )를 사용해서 만든 객체(Object)
- 클래스를 사용해서 객체를 만드는 과정을 인스턴스화라고 함
- 메모리에 할당 된 객체

4.필드
- 클래스 변수 (static variable) - 인스턴스를 생성하지 않고 사용가능 (공용변수)
- 인스턴스 변수 (instance variable) - 각 인스턴스마다 고유한 값을 가지고 있음
- 지역 변수 (Iocal variable) - 블록 내에서 사용할 수 있음 

5.메소드 
- 클래스에서 특정 작업을 수행하기 위한 명령문의 집합
- 오버로드 / 오버라이드 사용가능 

- 오버로드 : 함수의 이름은 같으나 매개 변수 , 린턴 타입 등의 특징을 다르게 설정할 수 있는 기능
- 오버라이드 : 부모 클래스의 메소드를 자식 클래스에서 재정의 한다.
